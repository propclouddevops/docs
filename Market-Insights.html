Market Insights — Price Trends Feature

Overview

Market Insights is a data feature that aggregates median property prices from historical and active listing data. It provides price trend analysis across multiple dimensions (location, category, bedroom count) and is displayed in three places:

  1. Dedicated Market Insights page (/market-insights) — full interactive experience
  2. Search results — price statistics widget at the bottom of results
  3. Listing detail page — contextual price trend widget

All price data is pre-computed by monthly scheduled tasks and served from an in-memory cache at runtime.

Currently enabled for the Mauritius portal (PropertyCloud) only.

Data Aggregation

What Gets Computed

The monthly task computes median prices across multiple dimension combinations:

flowchart TD O[Overall median by type and year] --- C[By category: Houses, Apartments, Land...] C --- SC[By sub-category: Villas, Townhouses...] SC --- P[By province/region] P --- T[By town] T --- S[By suburb] S --- B[By bedroom count: 1, 2, 3, 4, 5+] B --- X[Cross combinations e.g. Houses in Grand Baie]

How Medians Are Calculated

  1. All listing prices for a given year are loaded and normalized to a base currency using historical exchange rates
  2. Outlier removal: Only prices between the 5th and 95th percentiles are included
  3. The 50th percentile (true median) is computed from the filtered set
  4. Results are stored per combination of: year, sale/rental type, category, location, and bedroom count

Two Types of Median Data

TypeSourcePurpose
HistoricalAll past listings by yearPowers the year-over-year trend line chart
ActiveCurrently on-market listingsPowers the "current average price" displays and comparison tables

Both active medians include a price per square metre variant (using building area for properties, land area for land listings).

Display Locations

1. Market Insights Page (/market-insights)

The full interactive experience with multiple sections:

flowchart TD F[Filter Controls: Type, Category, Location] --> LC[Line Chart: Historical price trend] F --> BT[Bedroom Table: Median price by bedroom count] F --> CT[Category Table: Median by sub-category] F --> RT[Region Table: Median by province or town]
  • Users can filter by sale/rental type, category, and location (with autocomplete)
  • Changing filters triggers an AJAX call that returns updated chart data and price tables
  • Each table section can include AI-generated contextual commentary

2. Search Results Widget

Displayed at the bottom of search results when conditions are met:

  • A single location and/or category is selected
  • A mandate type (sale/rental) is active
flowchart LR SL[Single location selected] --> W[Price statistics by bedroom or sub-category] SC[Single category selected] --> W MT[Mandate type set] --> W W --> LK[Link to full Market Insights page]

Two display formats (configurable):

  • Table format: Rows with category/size and average price, each linking to filtered search results
  • Sentence format: Natural language cards with an "Explore historical price trends" link

3. Listing Detail Page Widget

Shows contextual price data for the specific listing being viewed:

  • Average price sentence: "The average price of [category] for [type] in [location] is [price]"
  • Historical trend line chart for that location/category combination
  • Price by bedroom table showing how this listing's bedroom count compares to the local median

Monthly Update Sequence

flowchart LR A[Update exchange rates] --> B[Compute active medians] B --> C[Compute active price per m2] C --> D[Compute group sums by views] D --> E[Compute historical medians current year] E --> F[Generate AI SEO blurbs] F --> G[Clear all caches]

AI-Generated SEO Content

For each valid combination of culture, type, category, and location, the system:

  1. Assembles the historical chart data, bedroom medians, sub-category medians, and region medians
  2. Sends these statistics as a prompt to an AI model
  3. Receives back contextual market commentary (historical trends, bedroom analysis, category breakdown, location insights)
  4. Stores the blurbs for display alongside the data tables

These are regenerated monthly as part of the scheduled update.
